## 1. 제품 개요

### 1.1 제품명

**SMIM** (Smart Media Insight Manager)

### 1.2 제품 설명

SMIM은 뉴스 기사 및 웹 글의 URL을 입력하면 본문을 자동으로 크롤링하고, Google Gemini AI를 활용하여 영어 학습에 유용한 추천 단어와 문장을 추출하는 서비스입니다. AI가 아티클 본문을 분석하여 CEFR 난이도를 자동 부여하고, 사용자의 영어 수준에 맞는 단어를 추출합니다. 추출된 단어는 개인 단어장에 저장하고, Spaced Repetition 기반의 퀴즈와 복습 알림을 통해 체계적으로 어휘를 학습할 수 있습니다.

### 1.3 핵심 가치

| 가치 | 설명 |
|------|------|
| **맥락 기반 학습** | 실제 뉴스/웹 글의 문맥 속에서 단어를 학습하여 암기 효율을 극대화합니다 |
| **AI 기반 큐레이션** | Gemini AI가 학습 가치가 높은 단어와 예문을 자동 선별하고, 아티클 난이도를 분석합니다 |
| **개인화된 단어장** | 사용자가 자유롭게 단어장을 구성하여 자기주도 학습을 지원합니다 |
| **스마트 복습** | Spaced Repetition 알고리즘과 다양한 퀴즈 유형으로 장기 기억 전환을 돕고, 이메일/카카오톡 알림으로 복습 시점을 놓치지 않도록 합니다 |
| **접근성** | URL 하나만 입력하면 복잡한 과정 없이 즉시 학습을 시작할 수 있습니다 |

### 1.4 현재 구현 현황

아래 기능은 이미 구현이 완료된 상태입니다.

- Spring Boot 4.0.1 기반 백엔드 애플리케이션
- OAuth2 소셜 로그인 (Google, Kakao) + JWT 인증/인가
- URL 크롤링 (Jsoup) 및 Article 저장
- 비동기 AI 단어 추출 (Google Gemini API) 및 ArticleVocabulary 저장
- User, Article, ArticleVocabulary 엔티티 및 기본 API
- 에러 코드 체계 (CO, AU, US, AR prefix)
- Redis 캐시 인프라, CORS 설정, 비동기 처리 설정

### 1.5 신규 개발 필요 항목

아래 기능은 이번 PRD를 통해 새로 정의하며, 개발이 필요합니다.

| 항목 | 우선순위 | 설명 |
|------|----------|------|
| 일반 로그인 (이메일 + 비밀번호) | **P0** | 소셜 로그인 외 이메일 기반 회원가입/로그인 |
| 일일 사용량 제한 (Rate Limiting) | **P0** | USER 역할 하루 5회, ADMIN 무제한 |
| 단어 선택 저장 / 전체 저장 | **P0** | 추출된 단어를 선택적 또는 일괄 저장 |
| 단어 재추출 | **P1** | 추출 결과가 불만족스러운 경우 재요청 |
| 커스텀 단어장 (Vocabulary Book) | **P0** | 사용자별 단어장 CRUD 및 단어 관리 |
| 난이도별 아티클 추천 (CEFR) | **P1** | AI 기반 CEFR 등급 분석 및 사용자 수준 맞춤 추천 |
| 학습 및 복습 (단어 퀴즈) | **P1** | Spaced Repetition 기반 퀴즈, 학습 통계 |
| 복습 알림 (이메일/카카오톡) | **P2** | 복습 시점에 이메일/카카오톡 알림 발송 |
| 난이도 기반 단어 추출 | **P1** | 크롤링 시 사용자 지정 난이도에 맞는 단어 추출 |

## 2. 문제 정의

### 2.1 현재 문제

1. **비효율적인 영어 학습**: 영어 뉴스나 웹 글을 읽을 때 모르는 단어를 일일이 사전에서 검색하고, 수동으로 정리하는 과정이 번거롭습니다.
2. **맥락 없는 단어 암기**: 기존 단어 학습 앱은 단어 목록을 제공할 뿐, 실제 글의 맥락 속에서 단어를 이해하는 데 한계가 있습니다.
3. **학습 자료 선별의 어려움**: 방대한 영어 콘텐츠 중 자신의 수준에 적합한 학습 단어를 골라내기 어렵습니다.
4. **체계적 관리 부재**: 학습한 단어를 주제별, 출처별로 체계적으로 관리할 수 있는 도구가 부족합니다.

### 2.2 해결 방안

SMIM은 AI 기반 자동 단어 추출과 개인화된 단어장 관리 기능을 통해 위 문제를 해결합니다. 사용자는 관심 있는 영어 콘텐츠의 URL만 입력하면, AI가 학습 가치가 높은 단어와 해당 문맥 속 예문을 자동으로 추출합니다. 추출된 단어는 개인 단어장에 저장하여 반복 학습에 활용할 수 있습니다.

## 5. 핵심 기능

### 5.1 인증 시스템

#### 5.1.1 소셜 로그인 (구현 완료)

현재 OAuth2 기반 Google 및 Kakao 소셜 로그인이 구현되어 있습니다.

- **지원 Provider**: Google, Kakao
- **인증 흐름**: OAuth2 Authorization Code Grant -> JWT Access Token + Refresh Token 발급
- **토큰 관리**: Access Token (단기), Refresh Token (장기), 토큰 갱신 API 제공

#### 5.1.2 일반 로그인 (신규 개발)

이메일과 비밀번호 기반의 회원가입 및 로그인 기능을 추가합니다.

**회원가입 요구 사항**:

| 필드 | 필수 여부 | 유효성 검증 규칙 |
|------|-----------|-----------------|
| 이메일 | 필수 | RFC 5322 형식, 중복 불가 |
| 비밀번호 | 필수 | 최소 8자, 영문 대/소문자 + 숫자 + 특수문자 포함 |
| 이름 | 필수 | 2~20자, 공백 불가 |

**보안 요구 사항**:

- 비밀번호는 BCrypt 해싱 후 저장
- 소셜 로그인으로 가입된 이메일과 중복 시 계정 연동 안내 메시지 반환
- 비밀번호 찾기/재설정 기능 제공 (이메일 인증 기반)

**API 설계**:

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/auth/signup` | 이메일 회원가입 |
| POST | `/api/auth/login` | 이메일 로그인 |
| POST | `/api/auth/password/reset-request` | 비밀번호 재설정 요청 (이메일 발송) |
| POST | `/api/auth/password/reset` | 비밀번호 재설정 (토큰 검증 후 변경) |

#### 5.1.3 Provider 확장

현재 `Provider` enum에 `LOCAL` 값을 추가하여 일반 로그인 사용자를 구분합니다.

```
Provider: GOOGLE, KAKAO, LOCAL
```

### 5.2 URL 크롤링 및 본문 추출

#### 5.2.1 기본 동작 (구현 완료)

사용자가 URL을 입력하면 Jsoup을 사용하여 웹 페이지의 본문을 크롤링하고, Article 엔티티로 저장합니다.

**크롤링 결과 저장 항목**:

| 필드 | 설명 |
|------|------|
| `title` | 페이지 제목 (`<title>` 또는 `og:title`) |
| `content` | 본문 텍스트 |
| `originalUrl` | 원본 URL |
| `sourceDomain` | 출처 도메인 (예: `bbc.com`) |

#### 5.2.2 크롤링 제약 사항

| 항목 | 제한 |
|------|------|
| 지원 프로토콜 | HTTP, HTTPS |
| 최대 본문 길이 | 50,000자 (초과 시 truncate) |
| 요청 Timeout | 10초 |
| 지원 언어 | 영문 콘텐츠 우선 (본문이 영어가 아닌 경우 경고 반환) |

#### 5.2.3 중복 URL 처리

동일 사용자가 이미 크롤링한 URL을 다시 요청하는 경우, 기존 Article을 반환하되 단어 재추출 옵션을 안내합니다.

### 5.3 AI 기반 추천 단어/문장 추출

#### 5.3.1 기본 동작 (구현 완료)

Article 저장 직후 비동기(`@Async`)로 Gemini API를 호출하여 단어를 추출합니다.

**추출 결과 항목**:

| 필드 | 설명 | 예시 |
|------|------|------|
| `word` | 추천 단어 | `unprecedented` |
| `definition` | 단어 정의 (영영 사전 기반) | `Never done or known before` |
| `contextSentence` | 기사 내 해당 단어가 사용된 문장 | `The country faces an unprecedented economic crisis.` |

#### 5.3.2 추출 설정

| 항목 | 값 |
|------|-----|
| 기본 추출 단어 수 | 10개 |
| 최소 추출 단어 수 | 5개 |
| 최대 추출 단어 수 | 20개 |
| 추출 기준 | 사용자 지정 난이도 또는 기본값 INTERMEDIATE (CEFR B1~B2) 기준으로 학습 가치가 높은 단어 |

#### 5.3.3 난이도 옵션 (신규)

크롤링 요청 시 사용자가 원하는 난이도를 지정할 수 있습니다. 지정하지 않을 경우 사용자 프로필의 `targetCefrLevel` 기반으로 자동 결정되며, 프로필에도 미설정 시 INTERMEDIATE가 기본값입니다.

| 난이도 | CEFR 범위 | 설명 |
|--------|-----------|------|
| `BEGINNER` | A1~A2 | 기초 수준 단어 추출 |
| `INTERMEDIATE` | B1~B2 | 중급 수준 단어 추출 (기본값) |
| `ADVANCED` | C1~C2 | 고급 수준 단어 추출 |

> **참고**: `ArticleCrawlRequest`에 선택적 `difficulty` 필드가 추가됩니다. Gemini 프롬프트에 난이도 파라미터를 포함하여 해당 수준에 적합한 단어를 추출합니다.

#### 5.3.4 추출 상태 관리

Article 엔티티의 `isCompleted` 필드를 활용하여 추출 상태를 관리합니다.

| 상태 | `isCompleted` | 설명 |
|------|:-------------:|------|
| 추출 진행 중 | `false` | 비동기 추출이 아직 완료되지 않은 상태 |
| 추출 완료 | `true` | 단어 추출이 성공적으로 완료된 상태 |

> **참고**: 추출 실패 시에도 Article은 유지되며, 사용자는 재추출을 요청할 수 있습니다.

### 5.4 일일 사용량 제한 (Rate Limiting) - 신규 개발

#### 5.4.1 제한 대상 API

| API | 한도 차감 |
|-----|----------|
| `POST /api/articles/crawl` | 1회 차감 |
| `POST /api/articles/{id}/re-extract` | 1회 차감 |

#### 5.4.2 구현 방식

- **저장소**: Redis를 활용한 사용자별 일일 카운터 (`rate_limit:{userId}:{date}`)
- **TTL**: 자정(KST) 기준으로 자동 만료 (최대 24시간)
- **검증 시점**: 크롤링/재추출 요청 처리 전 선행 검증
- **응답 헤더**: 잔여 횟수 정보를 응답 헤더에 포함

**응답 헤더 예시**:

```
X-RateLimit-Limit: 5
X-RateLimit-Remaining: 3
X-RateLimit-Reset: 2026-01-30T00:00:00+09:00
```

#### 5.4.3 한도 초과 응답

```json
{
  "success": false,
  "error": {
    "code": "AR004",
    "message": "일일 사용 한도를 초과하였습니다. (5/5)"
  }
}
```

### 5.5 단어 저장 (선택 저장 / 전체 저장) - 신규 개발

#### 5.5.1 기능 설명

AI가 추출한 단어를 사용자의 개인 단어장에 저장하는 기능입니다. 사용자는 원하는 단어만 선택하여 저장하거나, 추출된 전체 단어를 일괄 저장할 수 있습니다.

#### 5.5.2 저장 방식

| 방식 | 설명 | API |
|------|------|-----|
| **선택 저장** | 추출된 단어 목록에서 원하는 단어의 ID 목록을 지정하여 저장 | `POST /api/vocabulary-books/{bookId}/words/selective` |
| **전체 저장** | 특정 Article에서 추출된 모든 단어를 일괄 저장 | `POST /api/vocabulary-books/{bookId}/words/bulk` |

#### 5.5.3 저장 시 데이터 흐름

```
ArticleVocabulary (AI 추출 결과, 읽기 전용)
    ↓ 사용자 선택
VocabularyEntry (사용자 단어장에 저장된 단어)
```

- `ArticleVocabulary`는 Article에 종속된 AI 추출 결과로, 원본을 유지합니다.
- `VocabularyEntry`는 사용자가 개인 단어장에 저장한 단어로, 독립적으로 관리됩니다.
- 동일한 단어가 같은 단어장에 중복 저장되는 것을 방지합니다.

### 5.6 단어 재추출 - 신규 개발

#### 5.6.1 기능 설명

AI가 추출한 단어 목록이 만족스럽지 않은 경우, 사용자가 동일 Article에 대해 단어 재추출을 요청할 수 있습니다.

#### 5.6.2 동작 방식

1. 사용자가 재추출 API를 호출합니다.
2. 일일 사용량 한도를 확인합니다 (한도 1회 차감).
3. 기존 `ArticleVocabulary` 목록을 삭제합니다.
4. `isCompleted`를 `false`로 변경합니다.
5. 비동기로 Gemini API를 재호출하여 새로운 단어를 추출합니다.
6. 추출 완료 시 `isCompleted`를 `true`로 변경합니다.

#### 5.6.3 API

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/articles/{articleId}/re-extract` | 단어 재추출 요청 |

#### 5.6.4 제약 사항

- 이미 추출 진행 중인 Article에 대해서는 재추출 요청을 거부합니다 (`409 Conflict`).
- 재추출 시 기존에 사용자 단어장에 저장한 단어는 영향을 받지 않습니다.

### 5.8 난이도별 아티클 추천 - 신규 개발

#### 5.8.1 기능 설명

AI가 크롤링된 아티클의 본문을 분석하여 CEFR 난이도 등급을 자동으로 부여하고, 사용자의 영어 수준에 맞는 아티클을 추천하는 기능입니다.

#### 5.8.2 CEFR 등급 부여

Article 저장 시 Gemini API를 통해 본문의 CEFR 등급을 분석합니다. Article 엔티티에 `cefrLevel` 필드가 추가됩니다.

| CEFR 등급 | 설명 |
|-----------|------|
| `A1` | Beginner (입문) |
| `A2` | Elementary (초급) |
| `B1` | Intermediate (중급) |
| `B2` | Upper-Intermediate (중상급) |
| `C1` | Advanced (고급) |
| `C2` | Proficiency (최상급) |

#### 5.8.3 사용자 영어 수준 설정

사용자 프로필에 `targetCefrLevel` 필드를 추가하여, 본인의 영어 수준을 설정할 수 있습니다.

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `targetCefrLevel` | Enum (A1~C2) | X | 사용자 목표 CEFR 수준 (기본값: B1) |

#### 5.8.4 추천 API

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/articles/recommended` | 사용자 수준 기반 아티클 추천 목록 |
| GET | `/api/articles/trending` | 난이도별 트렌딩 아티클 목록 |

**추천 로직**:

- 사용자의 `targetCefrLevel`과 동일 또는 +-1 등급의 아티클을 우선 추천합니다.
- 정렬 기준: 최신순, 크롤링 횟수 순

**트렌딩 아티클**:

- 다수 사용자가 크롤링한 인기 URL을 난이도별로 분류하여 노출합니다.
- 쿼리 파라미터 `cefrLevel`로 특정 난이도 필터링이 가능합니다.

### 5.9 학습 및 복습 (단어 퀴즈) - 신규 개발

#### 5.9.1 기능 설명

단어장에 저장된 단어를 기반으로 다양한 유형의 퀴즈를 자동 생성하고, Spaced Repetition 알고리즘을 적용하여 효과적인 복습을 지원합니다.

#### 5.9.2 퀴즈 유형

| 유형 | 설명 | 예시 |
|------|------|------|
| **영->한 (4지선다)** | 영어 단어를 보고 올바른 한국어 뜻을 선택 | Q: `unprecedented` -> A: 전례 없는 |
| **한->영 (4지선다)** | 한국어 뜻을 보고 올바른 영어 단어를 선택 | Q: 전례 없는 -> A: `unprecedented` |
| **빈칸 채우기** | contextSentence에서 해당 단어를 빈칸으로 표시하고 직접 입력 | `The country faces an ___ economic crisis.` |

#### 5.9.3 퀴즈 세트

| 옵션 | 문제 수 |
|------|---------|
| Short | 10문제 |
| Medium | 20문제 |
| Full | 단어장 전체 |

#### 5.9.4 API

| Method | Endpoint | Description |
|--------|----------|-------------|
| POST | `/api/vocabulary-books/{bookId}/quiz` | 퀴즈 세트 생성 |
| POST | `/api/vocabulary-books/{bookId}/quiz/{quizId}/submit` | 퀴즈 결과 제출 |
| GET | `/api/vocabulary-books/{bookId}/quiz/history` | 퀴즈 히스토리 조회 |

**퀴즈 생성 요청**:

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `quizType` | Enum | O | `EN_TO_KO`, `KO_TO_EN`, `FILL_IN_THE_BLANK`, `MIXED` |
| `questionCount` | Enum | O | `SHORT(10)`, `MEDIUM(20)`, `FULL` |

#### 5.9.5 Spaced Repetition (에빙하우스 망각 곡선 기반)

VocabularyEntry에 학습 상태를 관리하기 위한 필드를 추가합니다.

| 필드 | 타입 | 설명 |
|------|------|------|
| `mastery` | Enum | 학습 상태: `NOT_STARTED`, `IN_PROGRESS`, `COMPLETED` |
| `nextReviewAt` | LocalDateTime | 다음 복습 예정 일시 |
| `reviewCount` | Integer | 총 복습 횟수 |
| `correctCount` | Integer | 정답 횟수 |

**복습 간격 알고리즘**:

| 단계 | 복습 간격 | 조건 |
|------|-----------|------|
| 1단계 | 1일 후 | 첫 학습 완료 |
| 2단계 | 3일 후 | 1단계 정답 |
| 3단계 | 7일 후 | 2단계 정답 |
| 4단계 | 14일 후 | 3단계 정답 |
| 5단계 | 30일 후 | 4단계 정답 |
| 완료 | - | 5단계 정답 시 `COMPLETED` |

- 정답 시 다음 단계로 진행합니다.
- 오답 시 1단계(1일 후)부터 다시 시작합니다.
- `mastery`가 `COMPLETED`가 되면 복습 대상에서 제외됩니다.

#### 5.9.6 학습 통계

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/learning/stats` | 종합 학습 통계 조회 |
| GET | `/api/learning/stats/daily` | 일별 학습 통계 |
| GET | `/api/learning/stats/weekly` | 주별 학습 통계 |
| GET | `/api/learning/stats/monthly` | 월별 학습 통계 |

**통계 항목**:

| 항목 | 설명 |
|------|------|
| 퀴즈 완료 수 | 일별/주별/월별 완료한 퀴즈 수 |
| 정답률 | 기간별 평균 정답률 |
| 연속 학습일 (스트릭) | 퀴즈를 1회 이상 완료한 연속 일수 |
| 단어 숙련도 분포 | `NOT_STARTED` / `IN_PROGRESS` / `COMPLETED` 비율 |
| 총 학습 단어 수 | 1회 이상 복습한 단어 수 |

### 5.10 복습 알림 (이메일/카카오톡) - 신규 개발

#### 5.10.1 기능 설명

복습 대상 단어가 있을 때 사용자가 설정한 채널(이메일, 카카오톡)로 알림을 발송하여 복습 시점을 놓치지 않도록 지원합니다.

#### 5.10.2 이메일 알림

- **기술**: Spring Boot Starter Mail (SMTP)
- **발송 시점**: 매일 아침 9시 (KST) 또는 사용자 설정 시간
- **이메일 내용**:
  - 오늘 복습할 단어 수
  - 복습 페이지 직접 링크
  - 최근 학습 통계 요약

#### 5.10.3 카카오톡 알림톡

- **기술**: 카카오톡 비즈니스 채널 알림톡 API
- **발송 시점**: 이메일과 동일 (사용자 설정 시간)
- **알림 내용**:
  - 복습 대상 단어 수
  - 앱 딥링크 (복습 페이지로 직접 이동)

#### 5.10.4 알림 설정 관리

| Method | Endpoint | Description |
|--------|----------|-------------|
| GET | `/api/notifications/settings` | 알림 설정 조회 |
| PUT | `/api/notifications/settings` | 알림 설정 수정 |

**알림 설정 항목**:

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `enabled` | Boolean | O | 알림 ON/OFF |
| `channel` | Enum | O | `EMAIL`, `KAKAO`, `BOTH`, `NONE` |
| `notificationTime` | LocalTime | X | 알림 수신 시간 (기본값: 09:00 KST) |

#### 5.10.5 알림 발송 조건

- 사용자의 `enabled`가 `true`이고 `channel`이 `NONE`이 아닌 경우에만 발송합니다.
- 복습 대상 단어(`nextReviewAt <= now()`)가 1개 이상 존재해야 합니다.
- 복습 대상 단어가 없는 경우 알림을 발송하지 않습니다.

### 5.11 난이도 기반 단어 추출 - 신규 개발

#### 5.11.1 기능 설명

크롤링 요청 시 사용자가 원하는 난이도를 지정하면, Gemini AI가 해당 수준에 적합한 단어를 추출하는 기능입니다.

#### 5.11.2 난이도 옵션

| 난이도 | CEFR 범위 | 추출 단어 특성 |
|--------|-----------|---------------|
| `BEGINNER` | A1~A2 | 기초 일상 어휘, 높은 빈도 단어 |
| `INTERMEDIATE` | B1~B2 | 중급 학술/시사 어휘, 중간 빈도 단어 |
| `ADVANCED` | C1~C2 | 고급 전문 어휘, 저빈도 고난이도 단어 |

#### 5.11.3 난이도 결정 우선순위

1. 크롤링 요청의 `difficulty` 파라미터 (최우선)
2. 사용자 프로필의 `targetCefrLevel` 기반 자동 매핑
3. 기본값: `INTERMEDIATE`

#### 5.11.4 API 변경 사항

`ArticleCrawlRequest`에 선택적 `difficulty` 필드를 추가합니다.

| 필드 | 타입 | 필수 | 설명 |
|------|------|:----:|------|
| `url` | String | O | 크롤링 대상 URL |
| `difficulty` | Enum | X | `BEGINNER`, `INTERMEDIATE`, `ADVANCED` |

## 부록

### A. 용어 정의

| 용어 | 정의 |
|------|------|
| Article | 사용자가 URL을 통해 크롤링한 뉴스 기사 또는 웹 글 |
| ArticleVocabulary | AI가 Article에서 추출한 추천 단어 (Article에 종속) |
| VocabularyBook | 사용자가 생성한 커스텀 단어장 |
| VocabularyEntry | 사용자 단어장에 저장된 개별 단어 항목 |
| Rate Limiting | 사용자별 일일 API 호출 횟수 제한 |
| Provider | 인증 제공자 (GOOGLE, KAKAO, LOCAL) |
| CEFR | Common European Framework of Reference for Languages. 유럽 공통 언어 능력 기준으로, A1(입문)부터 C2(최상급)까지 6단계로 언어 능력을 분류하는 국제 표준 |
| Spaced Repetition | 에빙하우스 망각 곡선 이론에 기반한 간격 반복 학습법. 복습 간격을 점진적으로 늘려 장기 기억 전환 효율을 극대화하는 학습 알고리즘 |
| 알림톡 | 카카오톡 비즈니스 채널을 통해 사용자에게 정보성 메시지를 발송하는 카카오 알림 서비스. 사전 등록된 템플릿 기반으로 동작 |
