# SMIM Backend 개선 제안서 (QA/PM 관점)

## 1. 문서 개요
- 대상: SMIM Backend API (PRD v2.0 기준)
- 작성일: 2026-02-13
- 목적: 구현 완료 기능에 대해 QA/PM 관점의 품질 개선 과제를 구조화하여 다음 스프린트 실행 항목으로 전환

## 2. 현재 기준선
- PRD 명시 API 누락 항목 보완 완료
- 단위 테스트 및 컨트롤러/서비스 중심 통합성 검증 코드 추가
- `./gradlew test`, `./gradlew build` 통과

## 3. QA 관점 개선 항목

### 3.1 P0 (즉시)
| ID | 항목 | 문제/리스크 | 개선안 | 완료 기준 |
|---|---|---|---|---|
| QA-P0-01 | 외부 네트워크 의존 테스트 분리 | BBC 크롤링 기반 테스트는 네트워크 상태에 따라 CI 변동 가능 | `@Tag("external")` 분리, 기본 CI에서는 mock 기반 테스트만 실행 | CI 기본 파이프라인에서 외부 호출 없는 안정적 test pass |
| QA-P0-02 | 보안 시나리오 테스트 확장 | Password reset 토큰 탈취/재사용/만료 시나리오 자동검증 부족 | 토큰 1회성, 만료, provider=LOCAL 제약, invalid token 시나리오 추가 | 보안 케이스 테스트 100% 통과 및 회귀 테스트 자동화 |
| QA-P0-03 | API 경로 호환성 회귀 테스트 | alias 경로(`/recommended`, `/words/*`, `PUT /settings`) 추가로 회귀 위험 | API contract 테스트(문서-엔드포인트 일치) 추가 | 문서 경로와 실제 매핑 diff가 0건 |

| ID | 항목 | 비즈니스 이슈 | 개선안 | KPI |

## 5. 권장 실행 순서 (2 Sprint)

### Sprint 1
- Password reset 메일 발송 플로우 완성
- API contract 회귀 테스트 자동화
- 외부 의존 테스트 분리 및 CI 안정화

### Sprint 2
- 트렌딩 스코어 모델 개선(행동 지표 기반)
- 학습 통계 KPI 대시보드 확장
- 성능 부하 테스트 자동화

## 6. 의사결정 필요 사항
- 트렌딩 산정 지표의 우선순위: 조회수 vs 저장수 vs 퀴즈완료율
- Password reset 메일 공급자 선택: SES/SendGrid/Naver Cloud
- 테스트 파이프라인 정책: 외부 연동 테스트의 nightly 분리 여부
